# 基础心电图 AI 生成 Prompt 模板（5 套）

> 使用 DeepSeek API `response_format: { type: "json_object" }` 模式，生成结构化教学病例。
> 每个 prompt 模板生成 20-24 个病例，共覆盖约 120 例。

---

## 模板 1：正常心电图入门（20 例）

### Prompt

```
你是心脏电生理教学专家。请生成 {count} 个基础心电图教学病例，覆盖正常心电图识别和基本测量。

要求：
1. 每个病例有标准 12 导联心电图的文字描述（模拟真实心电图所见）
2. 苏格拉底式提问：不直接给答案，通过 3-4 个递进问题引导判读
3. 覆盖知识点：P波形态、PR间期、QRS宽度、QTc、电轴目测、窦律确认

输出 JSON 格式（{count} 个对象的数组）：
```json
[
  {
    "title": "简洁的中文标题（<20字）",
    "category": "SVT",
    "difficulty": "基础",
    "description": "一句话概述（30字内）",
    "ecg_findings": {
      "details": [
        "心率: 75 bpm，窦律",
        "PR间期: 160ms，正常",
        "QRS宽度: 90ms，正常",
        "QTc: 420ms",
        "电轴: 约+45°（正常）",
        "P波形态: I/II导联直立，aVR倒置"
      ],
      "key_image_description": "简洁描述典型所见"
    },
    "question": "苏格拉底式起手问题（引导用户观察心电图）",
    "hint": "提示（不直接给答案）",
    "key_points": ["关键知识点1", "关键知识点2", "关键知识点3"],
    "learning_stages": [
      {"stage": 1, "ai_prompt": "第一层提问（观察层面）", "expected_answer": "期望的回答方向"},
      {"stage": 2, "ai_prompt": "第二层提问（解读层面）", "expected_answer": "期望的回答方向"},
      {"stage": 3, "ai_prompt": "第三层提问（推理/鉴别层面）", "expected_answer": "期望的回答方向"},
      {"stage": 4, "ai_prompt": "总结反馈", "expected_answer": "关键教学点"}
    ],
    "content_json": {
      "source": "心电学堂基础心电图系列"
    }
  }
]
```

知识点分布（{count} 例中）：
- 窦律确认：4 例（不同心率、P波形态变异）
- PR 间期：3 例（正常范围、一度AV阻滞的边界值）
- QRS 宽度：3 例（窄 QRS、临界宽度）
- QTc 测量：4 例（正常、边界值、不同心率下的纠正）
- 电轴判断：4 例（正常电轴、电轴偏左、电轴偏右的识别）
- 综合入门：2 例（多参数综合判断）
```

## 模板 2：心腔肥大 + 束支阻滞（20 例）

### Prompt

```
你是心脏电生理教学专家。请生成 {count} 个心电图教学病例，覆盖心腔肥大和束支阻滞的判读。

要求：
1. 每个病例有详细的 ECG 所见描述
2. 苏格拉底式递进提问
3. 覆盖知识点：左房异常/右房异常、左室肥大/右室肥大、RBBB/LBBB/LAFB/LPFB

输出 JSON 格式（同上模板的 schema）。

知识点分布（{count} 例中）：
- 左房异常（P mitrale）：3 例（二尖瓣型P波，V1导联终末负向增大）
- 右房异常（P pulmonale）：3 例（肺型P波，II/III/aVF导联高尖P波）
- 左室肥大（LVH）：4 例（Cornell/Sokolow-Lyon电压标准、ST-T继发改变）
- 右室肥大（RVH）：2 例（右室高电压+电轴右偏+V1导联R波优势）
- 完全性右束支阻滞（RBBB）：3 例（V1 rsR'形态、I/aVL/V5-V6宽S波）
- 完全性左束支阻滞（LBBB）：3 例（V1 QS型、I/aVL/V5-V6宽R波伴切迹）
- 左前分支阻滞（LAFB）：1 例（电轴左偏-60°、qR型I/aVL、rS型II/III/aVF）
- 左后分支阻滞（LPFB）：1 例（电轴右偏+120°、排除右室肥大）
```

## 模板 3：心肌缺血与梗死（24 例）

### Prompt

```
你是心脏电生理教学专家。请生成 {count} 个心电图教学病例，覆盖心肌缺血和心肌梗死的心电图判读。

要求：
1. 每个病例给出模拟的临床场景（年龄、主诉）+ ECG 所见描述
2. 苏格拉底式递进提问（观察→定位→血管推理）
3. 覆盖知识点：STEMI定位、NSTEMI、Wellens综合征、De Winter T波、后壁心梗、右室心梗

重要：输出必须标注「仅供教学使用，不构成临床决策建议」。

输出 JSON 格式（同上模板的 schema）。

知识点分布（{count} 例中）：
- 前壁 STEMI：4 例（前间壁 V1-V3、前壁 V2-V4、广泛前壁 V1-V6、高侧壁 I/aVL）
- 下壁 STEMI：4 例（单纯下壁、下壁+右室、下壁+后壁、下壁+前壁对应性ST段压低）
- 侧壁/后壁 STEMI：2 例（后壁 V7-V9、侧壁 I/aVL/V5-V6）
- NSTEMI：2 例（ST段压低为主+T波倒置，肌钙蛋白阳性）
- Wellens 综合征：2 例（LAD临界狭窄，胸痛缓解后T波特征性改变）
- De Winter T 波：1 例（LAD近端闭塞的特征性ST-T改变）
- STEMI 等位心电图：2 例（LBBB合并心梗的 Sgarbossa 标准）
- 心梗定位推理：4 例（根据ST段抬高导联反推罪犯血管：LAD/LCX/RCA）
- 心包炎 vs STEMI 鉴别：2 例
- 早期复极 vs STEMI 鉴别：1 例

每个病例提供：
- 临床场景（一句话）
- ECG 关键所见（3-6条）
- 问题链（观察→定位→血管→处理）
- 教学要点
```

## 使用指南

### 生成流程

```
1. 设置 {count} = 20（每次生成 20 例，分 6 批跑满 120 例）
2. 调用 DeepSeek API: POST /api/generate-book-case
3. response_format: { type: "json_object" }
4. 模型: deepseek-chat
5. max_tokens: 16384（确保一次返回 20 个对象的完整 JSON）
```

### 质量审核标准

生成后每例检查：
- [ ] 心电图描述在医学上准确
- [ ] 推理链逻辑自洽
- [ ] 不包含真实患者信息
- [ ] 标注「仅供教学」的免责声明
- [ ] category/difficulty 标签正确
- [ ] learning_stages 至少 3 个递进阶段

### 数据库写入

使用 `flattenCase()` 函数（`src/lib/case-utils.ts`）将 AI 返回的 JSON 映射为数据库字段，`product` 字段自动设为 `"ecg-academy"`。
